# foresee
强化学习
我构建了一个基于 PyTorch 和 PettingZoo 框架的多智能体强化学习 (MARL) 协作系统，主要用于解决多实体场景（如无人机群搜索、仓储机器人调度）下的去中心化路径分配与空间覆盖问题。

1. 项目解决的核心痛点：
传统的中央调度系统在面对海量实体时容易产生算力瓶颈和单点故障。本系统旨在解决在缺乏中央控制节点的情况下，如何让多个独立 Agent 通过局部观察进行自主决策，避免物理冲突（碰撞），并最终实现团队整体收益最大化的问题。

2. 核心逻辑流与多 Agent 协作：
系统采用了完全分布式的多 Agent 协作架构，底层基于独立 Q 学习 (Independent Q-Learning, IQL) 算法。每个 Agent 都配备了独立的多层感知机 (MLP) 决策网络和基于双向队列的经验回放池。

在运行工作流中：多个 Agent 在共享环境中并发执行动作。它们通过 Epsilon-Greedy 策略进行探索，收集环境状态、个体奖励以及碰撞惩罚。在长链条的周期性交互后，各 Agent 从自身的经验池中采样并利用深度 Q 网络（DQN）更新策略。系统通过这种去中心化的并行训练，让 Agent 逐渐涌现出协作行为（例如：主动避让队友、分散占领不同目标点）。目前该系统已在本地跑通，能够在 500 个训练回合内观察到明显的团队奖励收敛与协作涌现。
